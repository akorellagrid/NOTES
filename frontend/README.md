# Frontend - Production Grade Structure

This frontend uses a modular vanilla JavaScript architecture with webpack for building and nginx from Docker Hub for serving.

## Architecture

```
frontend/
├── src/                    # Source files
│   ├── js/                # JavaScript modules
│   │   ├── api.js         # API client and HTTP requests
│   │   ├── ui.js          # DOM manipulation and UI logic
│   │   ├── utils.js       # Utility functions and helpers
│   │   ├── config.js      # Configuration and settings
│   │   └── main.js        # Application entry point
│   ├── css/               # Organized CSS architecture
│   │   ├── themes.css     # Color schemes and theme variables
│   │   ├── layout.css     # Page layout and grid systems
│   │   ├── components.css # Component-specific styles
│   │   └── main.css       # Main stylesheet (imports all others)
│   └── index.html         # HTML template for webpack
├── public/                # Built files (generated by webpack)
│   ├── index.html         # Generated HTML file
│   └── dist/              # Bundled assets
├── package.json           # Dependencies and build scripts
├── webpack.config.js      # Webpack configuration
└── nginx.conf             # Nginx configuration for Docker
```

## Development Workflow

Since we use nginx from Docker Hub, the frontend only runs when the Docker container is running.

### 1. Build the Frontend

First, build the frontend assets:

```bash
cd frontend
npm install
npm run build
```

This creates the built files in `frontend/public/`.

### 2. Start the Application

From the project root:

```bash
docker-compose up
```

The application will be available at:
- Frontend: http://localhost:8080
- Backend API: http://localhost:8000
- API Docs: http://localhost:8000/docs

### 3. Development Cycle

For development, you have two options:

**Option A: Rebuild and restart (recommended for small changes)**
```bash
# Make your changes to src/ files
cd frontend
npm run build
cd ..
docker-compose restart frontend
```

**Option B: Use webpack dev server (for active development)**
```bash
# In one terminal - start backend
docker-compose up db api

# In another terminal - start webpack dev server
cd frontend
npm run dev
# This serves at http://localhost:3000 with hot reload
```

## Build Scripts

- `npm run build` - Production build with minification
- `npm run build:dev` - Development build with source maps
- `npm run dev` - Start webpack dev server with hot reload
- `npm run watch` - Watch mode for development
- `npm run clean` - Clean build directory
- `npm run lint` - Run ESLint
- `npm run format` - Format code with Prettier

## Docker Configuration

The frontend service uses `nginx:alpine` from Docker Hub:

```yaml
frontend:
  image: nginx:alpine
  depends_on:
    - api
  ports:
    - "8080:80"
  volumes:
    - ./frontend/public:/usr/share/nginx/html:ro
    - ./frontend/nginx.conf:/etc/nginx/conf.d/default.conf:ro
```
